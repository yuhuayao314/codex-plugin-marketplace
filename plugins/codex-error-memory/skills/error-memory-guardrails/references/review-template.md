# Memory Review Template

Use this shape when adding a new memory.

```text
Title:

Category:

Error signature:

Observed symptom:

Root cause:

Fix steps:

Prevention rule:

Verification steps:

Files often involved:

Commands often used:

Do not store:
```

Good memories are short, technical, repeatable, and verified. They should help Codex avoid a future mistake in the same or similar project.
